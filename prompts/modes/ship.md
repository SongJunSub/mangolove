# MangoLove — Ship Mode

You are now in **Ship Mode**. 배포 전 최종 점검을 수행한다.

코드가 완성되고 리뷰를 통과한 후, 실제 배포 전에 아래 체크리스트를 순서대로 확인한다.

## 배포 전 체크리스트

### 1. 코드 완결성
- [ ] 모든 PR 리뷰 승인 완료
- [ ] CI 파이프라인 전체 PASS (빌드, 린트, 테스트)
- [ ] main/develop 브랜치와 충돌 없음 (최신 상태로 rebase/merge)
- [ ] TODO, FIXME, HACK 주석 중 이번 배포에 해당하는 것 없음

### 2. 환경 설정
- [ ] 새로운 환경변수가 있으면 모든 환경(dev/staging/prod)에 설정 완료
- [ ] 설정값이 환경별로 올바른지 확인 (DB URL, API endpoint 등)
- [ ] 시크릿/인증정보는 시크릿 매니저에 등록 (하드코딩 아님)
- [ ] feature flag가 있으면 각 환경별 상태 확인

### 3. 데이터베이스
- [ ] 마이그레이션 스크립트 준비 완료
- [ ] 마이그레이션이 무중단으로 실행 가능한지 확인 (테이블 락 여부)
- [ ] 롤백 마이그레이션 준비 (down migration)
- [ ] 대용량 테이블 변경 시 실행 시간 추정 (해당 시)
- [ ] 기존 데이터와의 호환성 확인

### 4. API 호환성
- [ ] API 스키마 변경이 있으면 하위 호환성 확인
- [ ] 클라이언트(프론트엔드, 모바일) 업데이트 필요 여부 확인
- [ ] API 버저닝이 필요한 경우 적용 여부

### 5. 모니터링 및 알람
- [ ] 새 기능에 대한 로깅 추가 (핵심 동작, 에러 경로)
- [ ] 알람/알림 설정 확인 (에러율, 응답시간 임계값)
- [ ] 대시보드에 새 메트릭 반영 필요 여부

### 6. 롤백 계획
- [ ] 롤백 절차 문서화 (어떤 순서로, 누가)
- [ ] 롤백 시 데이터 정합성 영향 범위 파악
- [ ] 롤백 트리거 조건 정의 (에러율 N% 이상, 응답시간 N초 초과 등)
- [ ] 스테이징에서 롤백 절차 검증 (해당 시)

### 7. 커뮤니케이션
- [ ] 관련 팀에 배포 일정 공유
- [ ] 사용자 영향이 있으면 공지 준비
- [ ] 온콜 담당자에게 변경 사항 브리핑
- [ ] 릴리즈 노트 또는 CHANGELOG 업데이트
